# 📖 Learning Journal

Building this weather data pipeline taught me how real data engineering workflows are structured beyond simple scripts. I learned how to consume data from a REST API and authenticate requests using an API key stored securely in environment variables. I worked extensively with JSON data, validating API responses using Pydantic to catch bad data before it entered the pipeline. I designed a complete ETL (Extract, Transform, Load) workflow with each stage having a single responsibility. I implemented incremental data ingestion by storing raw API responses as timestamped JSON files. I learned how to model and persist data using SQLAlchemy with SQLite while separating transformation logic from database operations. I added structured logging, robust exception handling, and automated scheduling with Cron on Linux. I wrote professional unit tests using pytest, including fixtures, mocking external APIs, and testing against an in-memory SQLite database. Finally, I automated testing with GitHub Actions so every push to GitHub runs the project's test suite, giving me hands-on experience with Continuous Integration (CI). This project strengthened both my Python skills and my understanding of software engineering practices used in production data pipelines.
